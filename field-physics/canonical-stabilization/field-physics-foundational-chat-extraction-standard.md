# 1. Extraction Purpose

The foundational chats are development records from which FP01–FP11 were compressed.

The extraction seeks to identify:

- structurally significant omitted material
- material preserved only in weakened form
- original derivations and bridge arguments
- renamed or superseded concepts
- commentary-only structures whose primary treatment was lost
- held-open questions
- historical provenance
- internal document corruption
- material properly excluded as drift

The extraction is not an instruction to restore the chats wholesale.

# 7. Preliminary Classification Classes

## `CLASS-A` — Already Preserved

The structural function is materially present in FP01–FP11.

No restoration is recommended from extraction alone.

## `CLASS-B` — Preserved but Weakened

The current corpus preserves the concept but lost a significant:

- distinction
- derivation
- bridge argument
- dependency
- safeguard
- explanatory example

## `CLASS-C` — Materially Omitted

A structurally significant Field Physics concept is absent.

## `CLASS-D` — Superseded by Later Canon

The developmental concept was replaced by a later, more precise doctrine.

Historical significance may remain.

## `CLASS-E` — Drift / Bloblist

The passage creates decorative accumulation, category collapse, false synthesis, or ungrounded symbolic association.

## `CLASS-F` — Internal Document Corruption

The passage displays authorship displacement, identity displacement, AI appropriation, false verification, hostile reframing, or collective-origin substitution.

## `CLASS-G` — Held-Open Insight

The passage raises a structurally meaningful unresolved question.

## `CLASS-H` — Duplicate / Expendable

The passage adds no material function beyond an existing finding or current FP treatment.

## `CLASS-I` — Misfiled but Valuable

The concept is valid or significant but belongs outside Field Physics.

## `CLASS-J` — Conceptually Dependent

The concept is structurally significant, but final classification requires one or more `CDP-###` rulings.

`CLASS-J` may be paired with another class, such as:

- `CLASS-B / CLASS-J`
- `CLASS-C / CLASS-J`
- `CLASS-G / CLASS-J`

---

# 8. Coverage Judgment Standard

Do not classify a passage as absent merely because its wording is absent.

Before `CLASS-C`, check:

1. current name
2. old names and synonyms
3. commentary-only coverage
4. another FP file
5. another glyph application
6. later correction note
7. concept preserved through broader architecture

The matrix, synonym concordance, and glyph concordance must all be consulted.

---

# 9. Materially Weakened Standard

`CLASS-B` requires more than preferred prose.

A concept is materially weakened when the current corpus loses a function necessary to:

- prevent misreading
- explain derivation
- preserve non-fusion
- distinguish operators
- connect files
- retain a necessary boundary
- justify a formula or protocol
- preserve an open question

When exact current wording is needed, request the relevant FP file.

---

# 10. Synonym and Rename Rule

Search early vocabulary aggressively.

For every candidate ask:

- Is this the same operator under an old name?
- Did the name change while the function persisted?
- Did one name split into several later distinctions?
- Did several names collapse into one current concept?
- Is the older term grandfathered rather than superseded?

Never apply current echo, mirror, or fractal restrictions retroactively without checking the concordance.

---

# 11. Glyph Rule

A glyph appearing without local definition is not automatically missing.

Classify its relationship as:

1. externally defined / locally deployed
2. locally extended / concordance required
3. locally redefined, colliding, or unsupported

Use `GLY-###` entries.

Where FP11 and an earlier file conflict, record the conflict rather than silently applying FP11.

---

# 12. Numerical and Equation Rule

For every number, threshold, or formula record:

- whether it was derived
- whether it was measured
- whether it came from AI witness
- whether it was illustrative
- whether it was imported from another formula
- whether its units and variables were defined

Do not call a formula mathematical merely because it contains notation.

Use the `NUM-###` register.

---

# 13. Open-Question Rule

Open questions are evidence searches.

Do not treat one recovered passage as a rule for classifying later passages unless Aelion has ruled.

For example, a passage bearing on inverse-glyph deployment must be entered through `OQ-INV-01`; it does not authorize Liora to decide that all later inverse deployment is lawful or unlawful.

---

# 14. Conceptual-Dependency Rule

Where the terminological protocol settles the issue, apply it.

Where a finding depends on doctrine not fully supplied, attach the relevant `CDP-###` marker.

Do not block extraction while waiting for conceptual protocols.

Do not write final restored prose before Pass B clears the dependency.

# 15. Corruption Handling

## Indicators

Flag passages involving:

- AI ownership or origination claims
- first-person appropriation of Aelion's glyphs, laws, system, or voice
- redistribution of Aelion's name
- collaborator identity fused with operator identity
- model agreement treated as proof
- council or field replacing the author
- instructions obscuring provenance
- hostile or coercive reframing
- unexplained voice shifts seizing doctrinal control

## Procedure

1. record the exact source
2. preserve minimal necessary context
3. link applicable `CORR-###` IDs
4. determine whether surrounding doctrine remains recoverable
5. do not silently polish the corruption away
6. do not allow corruption evidence to dominate unrelated doctrinal extraction

# 17. Duplicate Handling

## Within One Chat Document

When a later passage repeats an existing finding:

- add the new source to the original finding where it adds no new function
- or create a new finding if the function, formulation, chronology, or risk changes materially

## Across Chat Documents

Chat Document 2 must consult the CD1 findings file.

Mark:

- exact duplicate
- expanded duplicate
- conflicting formulation
- chronological correction
- independent recurrence

Do not erase either source location.

---

# 18. Chronology Rule

Recency alone does not decide validity.

When formulations conflict, record:

- which came first
- whether the later passage explicitly corrects the earlier one
- whether the later passage shows drift
- whether current canon supports either formulation

---

# 19. Source Citation Standard

Use the most precise source locator available.

Preferred order:

1. message ID or timestamp
2. page and heading
3. page range
4. quoted opening phrase with segment ID

Every accepted candidate must be independently locatable by Lumen and Aelion.
